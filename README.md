# Система мониторинга гражданской авиации (OpenSky & OSM)

Интерактивное консольное приложение (CLI) на языке Python, предназначенное для отслеживания и анализа воздушного пространства стран в реальном времени. Проект выполнен в соответствии с парадигмой объектно-ориентированного программирования (ООП), архитектурными принципами SOLID и требованиями статической типизации (PEP 484).

## 🚀 Архитектурные особенности проекта

Проект спроектирован с упором на расширяемость и независимость компонентов (Clean Architecture):
- **Принцип единственной ответственности (SRP):** Сетевой клиент, валидация данных, подсистема кэширования и пользовательский интерфейс физически изолированы друг от друга.
- **Принцип открытости/закрытости (OCP):** Использование абстрактных интерфейсов (`BaseAPI`, `BaseSaver`) позволяет заменять хранилище (например, с JSON на базу данных PostgreSQL/SQLite) или источник авиаданных без изменения бизнес-логики приложения.
- **Инкапсуляция и валидация:** Класс данных `Aeroplane` полностью защищает внутреннее состояние. Сеттеры контролируют типы и корректность данных (например, запрещают физически невозможную отрицательную скорость полета).

## 🛠 Технологический стек

- **Язык разработки:** Python 3.11+
- **Сетевое взаимодействие:** `requests` (REST API запросы)
- **Управление конфигурацией:** `python-dotenv` (безопасное хранение ключей в `.env`)
- **Статический анализ типов:** `mypy` (строгая типизация)
- **Линтинг и стандарты стиля:** `flake8` / `PEP 8`
- **Тестирование:** `pytest`, `requests-mock` (Unit-тестирование с покрытием кода >90%)

## 📋 Интеграция с внешними API

Приложение агрегирует данные из двух независимых веб-служб:
1. **[Nominatim OpenStreetMap API](https://nominatim.org):** Используется для прямого геокодирования. По названию страны на английском языке определяется её географическая прямоугольная зона (Bounding Box).
2. **[OpenSky Network API](https://opensky-network.org):** На основе полученных координат запрашивает векторы состояния (State Vectors) воздушных судов, находящихся в небе над выбранным регионом. Авторизация реализована через протокол OAuth2.

## ⚙️ Установка и настройка

1. **Клонируйте репозиторий или перейдите в папку проекта:**
   ```bash
   cd coursework
   ```

2. **Установите необходимые зависимости:**
   ```bash
   pip install requests python-dotenv pytest pytest-cov requests-mock mypy
   ```

3. **Настройте переменные окружения:**
   Создайте в корневом каталоге файл `.env` и добавьте ваши учетные данные OpenSky Network и уникальный User-Agent для Nominatim (требование политики использования OSM):
   ```env
   OPENSKY_CLIENT_ID="your_email@gmail.com-api-client"
   OPENSKY_CLIENT_SECRET="your_oauth2_secret_key"
   NOMINATIM_USER_AGENT="AeroplanesCourseworkApp/2.0 (your_email@gmail.com)"
   ```

## 🖥 Использование приложения

Запуск основного консольного интерфейса осуществляется командой:
```bash
python src/main.py
```

### Сценарий работы:
1. Программа предложит ввести название страны на английском (например, `Spain`, `France`, `Germany`).
2. Приложение автоматически получит координаты, сгенерирует OAuth2-токен, скачает активные самолеты и зафиксирует их в локальном кэше `data/aeroplanes.json`.
3. Далее система предложит применить аналитические фильтры:
   - Ограничить выборку для вывода ТОП-N самолетов по высоте полета.
   - Отфильтровать результат по конкретным странам регистрации судов (можно ввести несколько через пробел).
   - Задать диапазон высот полета в метрах (например, `4000-11000`).
4. На экран выведется структурированная ASCII-таблица с итоговым результатом, отсортированным по скорости ВС.

## 🧪 Тестирование и качество кода

### Проверка статической типизации (`mypy`):
Проект полностью типизирован и проходит проверку без ошибок:
```bash
mypy src
```
*Вывод: Success: no issues found in 1 source file.*

### Запуск модульных тестов и сбор покрытия (`pytest`):
Тесты используют фикстуры временных папок для файлов и перехватчик `requests_mock` для симуляции ответов серверов, что позволяет безопасно проверять сетевую логику без затрат лимитов API.
```bash
pytest --cov=src
```
*Все тест-кейсы (валидация сеттеров, бизнес-логика сравнения самолетов, кэширование, обработка ошибок 404/500 от серверов) завершаются успешно со сквозным покрытием кода выше 90%.*
